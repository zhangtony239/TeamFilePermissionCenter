# 流程/时序图（集中展示）

本文件集中放置关键业务流程与时序图，供说明书引用。

## 1) 登录与鉴权时序

```mermaid
sequenceDiagram
  participant U as 浏览器
  participant N as Nginx
  participant A as API（后端）
  participant DB as 数据库

  U->>N: POST /auth/login
  N->>A: 转发请求
  A->>DB: 校验账号/密码/状态
  DB-->>A: 校验结果
  alt 通过
    A-->>U: 返回 Session/JWT
  else 失败
    A-->>U: 401/403
  end
```

## 2) 文件树浏览流程（含个人区开关）

```mermaid
flowchart TD
  S[进入项目文件页] --> T{开关：显示个人区?}
  T -->|是| Q1[请求目录树 include_personal=true]
  T -->|否| Q2[请求目录树 include_personal=false]

  Q1 --> AUTH[鉴权]
  Q2 --> AUTH

  AUTH --> SAN[路径沙箱<br/>规范化/防穿越]
  SAN --> VIS[可见性过滤<br/>LIST + 个人区规则]
  VIS --> OUT[返回目录树/列表]
  OUT --> AUD[写审计<br/>action=LIST]
```

## 3) PDF 预览时序（Range + 低负载）

```mermaid
sequenceDiagram
  participant U as 浏览器（PDF.js）
  participant N as Nginx
  participant A as API（文件服务）
  participant P as 权限服务
  participant S as 存储（磁盘）
  participant L as 审计

  U->>N: GET /files/preview?path=...
  N->>A: 转发
  A->>P: 校验 PREVIEW 权限
  P-->>A: allow/deny
  alt allow
    A->>S: Range 读取(按需分片)
    S-->>A: 返回分片
    A->>L: 记录审计(PREVIEW)
    A-->>U: 分片响应(支持 Range)
    U-->>U: 浏览器本地渲染/缓存
  else deny
    A->>L: 记录审计(DENY)
    A-->>U: 403
  end
```

## 4) 下载时序（按权限控制）

```mermaid
sequenceDiagram
  participant U as 浏览器
  participant A as API
  participant P as 权限服务
  participant S as 存储
  participant L as 审计

  U->>A: GET /files/download?path=...
  A->>P: 校验 DOWNLOAD
  alt allow
    A->>S: 流式读取
    S-->>A: bytes...
    A->>L: 记录审计(DOWNLOAD)
    A-->>U: attachment 响应
  else deny
    A->>L: 记录审计(DENY)
    A-->>U: 403
  end
```

## 5) 上传/覆盖更新流程（触发版本与索引）

```mermaid
flowchart TD
  U[用户上传/覆盖] --> AUTH[鉴权 + UPLOAD/UPDATE]
  AUTH --> SAN[路径沙箱/规范化]
  SAN --> W[写入存储]
  W --> V[写 FileVersion 事件]
  V --> M[更新 FileAsset 元信息（hash、size、mtime）]
  M --> J[创建 IndexJob（异步）]
  J --> AUD[写审计 UPLOAD/UPDATE]
```

## 6) 删除/回收站/恢复流程

```mermaid
flowchart TD
  D[删除请求] --> AUTH[鉴权 + DELETE]
  AUTH --> BIN[写入 RecycleItem<br/>软删除（从列表隐藏）]
  BIN --> V[写 FileVersion event=DELETE]
  V --> AUD[写审计 DELETE]

  R[恢复请求] --> AUTH2[鉴权 + RESTORE]
  AUTH2 --> RES[从回收站恢复<br/>恢复原路径/或新路径]
  RES --> V2[写 FileVersion event=RESTORE]
  V2 --> AUD2[写审计 RESTORE]

  P[到期清理任务] --> PURGE[物理删除/归档]
  PURGE --> AUD3[写审计 PURGE]
```

## 7) 权限变更时序（ACL 管理）

```mermaid
sequenceDiagram
  participant Admin as 管理员/项目管理员
  participant A as API
  participant DB as 数据库
  participant L as 审计

  Admin->>A: POST /acl/grant|revoke（目标路径/动作/主体）
  A->>DB: 写 FileACL (ALLOW/DENY)
  DB-->>A: ok
  A->>L: 记录审计(ACL_CHANGE)
  A-->>Admin: 200

  Note over A: 后续访问立即基于最新 ACL 生效
```
