# 部署指南（无反向代理，IP:端口访问）

本文档用于在一台 Ubuntu 服务器上部署本项目（前端 + 后端 + MinIO 文件存储），并通过 `http://<SERVER_IP>:端口` 的方式访问。

- 代码目录（约定）：
  - 后端：`/home/ubuntu/server`
  - 前端：`/home/ubuntu/web`
- 访问端口（默认）：
  - 前端：`5173`
  - 后端 API：`8000`
  - MinIO API：`9000`
  - MinIO Console：`9001`（可选）

> 注意：文档中所有 `<SERVER_IP>` 需要替换成你的服务器公网 IP。

---

## 1. 服务器安装与基础准备

### 1.1 安装依赖（Docker / Python / Node / UFW）

```bash
sudo apt update

# 常用工具
sudo apt install -y ca-certificates curl git ufw

# Python
sudo apt install -y python3 python3-venv python3-pip

# Node（用于前端构建）
sudo apt install -y nodejs npm

# Docker
sudo apt install -y docker.io
sudo systemctl enable --now docker
```

### 1.2（可选但推荐）允许 ubuntu 用户直接运行 docker

```bash
sudo usermod -aG docker ubuntu
# 重新登录 SSH 使权限生效；或临时：newgrp docker
```

### 1.3（可选但推荐）Docker 镜像加速

如果 `docker pull` 出现超时（如 Docker Hub 连接慢），可配置镜像源：

```bash
sudo mkdir -p /etc/docker

sudo tee /etc/docker/daemon.json > /dev/null <<'EOF'
{
  "registry-mirrors": [
    "https://mirror.ccs.tencentyun.com",
    "https://docker.mirrors.ustc.edu.cn",
    "https://hub-mirror.c.163.com"
  ],
  "dns": ["223.5.5.5", "114.114.114.114", "8.8.8.8"]
}
EOF

sudo systemctl daemon-reload
sudo systemctl restart docker
```

## 2. 开放端口（UFW + 云安全组）

你需要在“云安全组”放行入站端口；若服务器启用 UFW，也需要放行：

```bash
sudo ufw allow 22/tcp
sudo ufw allow 8000/tcp
sudo ufw allow 5173/tcp
sudo ufw allow 9000/tcp
sudo ufw allow 9001/tcp
sudo ufw --force enable
sudo ufw status verbose
```

## 3. 部署 MinIO（Docker）

本项目的上传/下载/预览使用 MinIO（S3 兼容存储）。

### 3.1 启动 MinIO 容器

```bash
mkdir -p /home/ubuntu/minio-data

docker pull minio/minio:latest

docker rm -f tfpc-minio 2>/dev/null || true

docker run -d --name tfpc-minio --restart=always \
  -p 9000:9000 -p 9001:9001 \
  -v /home/ubuntu/minio-data:/data \
  -e MINIO_ROOT_USER=minioadmin \
  -e MINIO_ROOT_PASSWORD=minioadmin123 \
  -e MINIO_SERVER_URL=http://<SERVER_IP>:9000 \
  -e MINIO_BROWSER_REDIRECT_URL=http://<SERVER_IP>:9001 \
  minio/minio:latest server /data --console-address ":9001"
```

> 账号密码（示例）：`minioadmin / minioadmin123`。生产环境请自行修改。

### 3.2 验证 MinIO

```bash
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"
curl -I http://127.0.0.1:9000/
curl -I http://127.0.0.1:9001/
```

- MinIO Console：`http://<SERVER_IP>:9001`

## 4. 部署后端（Django + Gunicorn + systemd 常驻）

### 4.1 安装 Python 依赖

```bash
cd /home/ubuntu/server
python3 -m venv .venv
source .venv/bin/activate

pip install -U pip
pip install -r requirements.txt
```

### 4.2 创建后端环境变量文件 `/home/ubuntu/server/.env`

```bash
cat > /home/ubuntu/server/.env << 'EOF'
DJANGO_DEBUG=0
DJANGO_SECRET_KEY=please-change-me
DJANGO_ALLOWED_HOSTS=*

# 前端来源（无反代）
CORS_ALLOWED_ORIGINS=http://<SERVER_IP>:5173

# MinIO（本机运行）
MINIO_ENDPOINT=127.0.0.1:9000
# 预签名 URL 给浏览器访问必须是公网 IP
MINIO_PUBLIC_ENDPOINT=<SERVER_IP>:9000
MINIO_ACCESS_KEY=minioadmin
MINIO_SECRET_KEY=minioadmin123
MINIO_BUCKET=tfpc
MINIO_SECURE=0
EOF
```

### 4.3 数据库迁移 + 创建超级管理员

```bash
cd /home/ubuntu/server
source .venv/bin/activate

python manage.py migrate
python manage.py createsuperuser
```

> 如果忘记密码：`python manage.py changepassword <username>`

### 4.4 配置 systemd 服务（tfpc-server）

```bash
sudo tee /etc/systemd/system/tfpc-server.service > /dev/null << 'EOF'
[Unit]
Description=TFPC Django API (gunicorn)
After=network.target docker.service
Requires=docker.service

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/home/ubuntu/server
EnvironmentFile=/home/ubuntu/server/.env
ExecStart=/home/ubuntu/server/.venv/bin/gunicorn tfpc.wsgi:application --bind 0.0.0.0:8000 --workers 2 --timeout 120
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable --now tfpc-server
sudo systemctl status tfpc-server --no-pager
```

验证后端（401/403 也算正常，说明服务在）：

```bash
curl -I http://127.0.0.1:8000/api/v1/
```

## 5. 部署前端（build + 静态服务 + systemd 常驻）

### 5.1 前端构建（写入后端 API 地址）

```bash
cd /home/ubuntu/web
npm install

# 关键：build 时写入 API_BASE
VITE_API_BASE=http://<SERVER_IP>:8000/api/v1 npm run build
```

> 若构建报 `Permission denied`，优先执行：
>
> ```bash
> sudo chown -R ubuntu:ubuntu /home/ubuntu/web
> rm -rf node_modules package-lock.json
> npm install
> ```

### 5.2 静态服务（推荐使用 serve 支持 SPA 路由）

```bash
sudo npm i -g serve
which serve
```

创建 systemd 服务：

```bash
sudo tee /etc/systemd/system/tfpc-web.service > /dev/null << 'EOF'
[Unit]
Description=TFPC Web (static)
After=network.target

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/home/ubuntu/web
ExecStart=/usr/local/bin/serve -s /home/ubuntu/web/dist -l 5173
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable --now tfpc-web
sudo systemctl status tfpc-web --no-pager
```

访问前端：
- `http://<SERVER_IP>:5173`

## 6. 常用检查与排障

### 6.1 端口监听

```bash
sudo ss -lntp | egrep '(:5173|:8000|:9000|:9001)\b' || true
```

### 6.2 服务状态与日志

```bash
sudo systemctl status tfpc-server --no-pager
sudo systemctl status tfpc-web --no-pager

journalctl -u tfpc-server -n 100 --no-pager
journalctl -u tfpc-web -n 100 --no-pager
```

### 6.3 MinIO 容器日志

```bash
docker logs --tail 100 tfpc-minio
```
