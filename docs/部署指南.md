# 部署指南（docker-compose，仅暴露 127.0.0.1:5173）

本文档用于在任意装有 Docker 的主机上部署本项目（前端 + 后端 + MinIO 文件存储），通过 `docker compose` 一键拉起，组件之间以**容器名直连**，对外**仅暴露** `http://127.0.0.1:5173` 一个 Web 入口。

## 0. 架构总览

```
浏览器 ──http──> 127.0.0.1:5173 (nginx 容器 tfpc-web)
                        │
                        ├── /            → SPA 静态页面（前端构建产物）
                        ├── /api/        → server:8000（Django/DRF）
                        ├── /admin/      → server:8000（Django Admin）
                        ├── /static/     → 共享卷（server collectstatic 产物）
                        └── /tfpc/       → minio:9000（文件下载/预览，预签名 URL）

容器间直连（仅 docker 内网，不对外）：
  - web   → server:8000
  - web   → minio:9000
  - server→ minio:9000
```

关键设计：

- **前端与后端同源**：前端 build 时注入 `VITE_API_BASE=/api/v1`，浏览器所有 API 请求都走 `127.0.0.1:5173/api/...`，由 nginx 反代到 `server:8000`。因此**无需 CORS、无需知道服务器 IP**。
- **MinIO 不直接对外暴露**：MinIO 仅监听容器内 `9000`。文件下载/预览由后端生成预签名 URL（指向 `127.0.0.1:5173`），再由 nginx 把 `/tfpc/` 反代到 `minio:9000`。
- **预签名 URL 的 Host 一致性**：MinIO 预签名签名会校验 `Host` 头，nginx 反代时保留浏览器原始 `Host`（`127.0.0.1:5173`），与后端 `MINIO_PUBLIC_ENDPOINT` 一致，避免 `SignatureDoesNotMatch`。

对应代码改动：

- [`server/core/storage.py`](../server/core/storage.py) 分离了「内部上传 client（`MINIO_ENDPOINT`）」与「对外预签名 client（`MINIO_PUBLIC_ENDPOINT`）」。
- [`server/tfpc/settings.py`](../server/tfpc/settings.py) 新增 `STATIC_ROOT`（collectstatic）与 `CSRF_TRUSTED_ORIGINS`。
- [`web/Dockerfile`](../web/Dockerfile) 多阶段构建，注入 `VITE_API_BASE=/api/v1`。
- [`web/nginx.conf`](../web/nginx.conf) 统一反代入口。

## 1. 准备宿主机

### 1.1 安装 Docker 与 Compose

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install -y ca-certificates curl git
# 安装 Docker Engine（含 Compose 插件）
curl -fsSL https://get.docker.com | sudo sh
sudo systemctl enable --now docker

# 允许当前用户直接运行 docker（可选，需重新登录生效）
sudo usermod -aG docker $USER
```

验证：

```bash
docker version
docker compose version
```

### 1.2（可选）Docker 镜像加速

若 `docker pull` 超时，可配置镜像源：

```bash
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json > /dev/null <<'EOF'
{
  "registry-mirrors": [
    "https://mirror.ccs.tencentyun.com",
    "https://docker.mirrors.ustc.edu.cn",
    "https://hub-mirror.c.163.com"
  ],
  "dns": ["223.5.5.5", "114.114.114.114", "8.8.8.8"]
}
EOF
sudo systemctl daemon-reload
sudo systemctl restart docker
```

### 1.3 放行端口（仅本机访问，通常无需对外）

由于 Web 只绑定 `127.0.0.1:5173`，**默认不监听公网**，无需在云安全组放行 5173。

- 若希望仅本机浏览器访问：无需任何防火墙配置。
- 若希望局域网/公网访问：需把 `docker-compose.yml` 中 `web` 服务的端口映射改为 `"5173:5173"`（监听所有网卡），并在云安全组/UFW 放行 5173。**注意：这会让 5173 对外可访问，请自行评估安全风险（建议前置 HTTPS 反代或 SSH 隧道）。**

## 2. 配置环境变量

在项目根目录复制示例配置并按需修改：

```bash
cp .env.docker.example .env
```

必改项：

| 变量 | 说明 | 示例 |
|---|---|---|
| `DJANGO_SECRET_KEY` | 生产务必改为随机长字符串 | 用 `python -c "import secrets;print(secrets.token_urlsafe(50))"` 生成 |
| `MINIO_ROOT_PASSWORD` | MinIO 控制台/Root 密码 | 至少 8 位 |
| `DJANGO_SUPERUSER_USERNAME/PASSWORD` | 首次启动自动创建的超管 | 创建后可清空 |

其余变量（`MINIO_ENDPOINT=minio:9000`、`MINIO_PUBLIC_ENDPOINT=127.0.0.1:5173`、`VITE_API_BASE=/api/v1`）已为同源方案预设，**一般无需改动**。

## 3. 一键启动

在项目根目录（`docker-compose.yml` 所在目录）执行：

```bash
docker compose up -d --build
```

首次构建会拉取基础镜像、安装依赖、构建前端，耗时较长。完成后查看状态：

```bash
docker compose ps
```

预期看到 `tfpc-minio`、`tfpc-server`、`tfpc-web` 三个容器均为 `Up`。

查看日志：

```bash
docker compose logs -f web
docker compose logs -f server
```

后端容器启动时会自动执行 `collectstatic` 与 `migrate`；若设置了 `DJANGO_SUPERUSER_*`，会自动创建超管。

## 5. 常用运维命令

```bash
# 启动
docker compose up -d --build

# 停止（保留数据）
docker compose down

# 停止并删除数据卷（谨慎！会清空 MinIO 文件与数据库）
docker compose down -v

# 查看实时日志
docker compose logs -f

# 进入后端容器执行 Django 管理命令
docker compose exec server python manage.py createsuperuser
docker compose exec server python manage.py migrate
docker compose exec server python manage.py backup_system

# 进入 MinIO 容器
docker compose exec minio sh
```

数据持久化卷：

- `tfpc_minio-data`：MinIO 文件数据
- `tfpc_tfpc-db`：SQLite 数据库
- `tfpc_tfpc-backups`：后端备份输出
- `tfpc_django-static`：collectstatic 静态资源（web 只读挂载）

## 7. 排障

### 7.1 前端能打开但登录失败（401 / 网络错误）

确认前端构建时 `VITE_API_BASE` 为 `/api/v1`（同源）。检查 `.env`：

```bash
grep VITE_API_BASE .env
# 应为：VITE_API_BASE=/api/v1
```

若改动过需重新 build：`docker compose up -d --build web`。

### 7.2 文件下载/预览 403 SignatureDoesNotMatch

预签名 URL 的 `Host` 必须与后端 `MINIO_PUBLIC_ENDPOINT` 一致。确认：

- `.env` 中 `MINIO_PUBLIC_ENDPOINT=127.0.0.1:5173`
- `web/nginx.conf` 的 `/tfpc/` 块 `proxy_set_header Host $http_host;`（保留原始 Host，不要被改成 `minio:9000`）。

若改了 `MINIO_BUCKET`，需同步修改 nginx 中 `location /tfpc/` 的前缀。

### 7.3 端口被占用

`5173` 被占用时改 `.env` 无效，需改 `docker-compose.yml` 中 `web` 的端口映射，例如 `"127.0.0.1:8080:5173"`，并把 `MINIO_PUBLIC_ENDPOINT` 同步改为 `127.0.0.1:8080`。

### 7.4 想让 MinIO Console 可用

默认未暴露 9001。临时查看可用：

```bash
docker compose exec -d minio minio server /data --console-address ":9001"
# 或临时端口映射（需改 compose）：
# ports: ["127.0.0.1:9001:9001"]
```

### 7.5 后端容器日志

```bash
docker compose logs --tail 200 server
```
