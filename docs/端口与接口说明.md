# 端口与接口说明（docker-compose，仅暴露 5173）

本文档说明 docker-compose 部署后各端口/路由用途，以及经 `127.0.0.1:5173` 统一入口暴露的路径。

> 采用同源方案：浏览器只访问 `http://127.0.0.1:5173`，所有后端 API、Django Admin、MinIO 文件下载/预览均由 nginx 在该端口内部反代，组件间走容器名直连，**不对外暴露其他端口**。

---

## 1. 端口用途总览

### 1.1 对外暴露（宿主机）

| 端口 | 绑定 | 服务 | 访问方式 | 备注 |
|---:|---|---|---|---|
| 5173 | `127.0.0.1:5173` | nginx（前端 + 反代） | `http://127.0.0.1:5173` | **唯一对外入口**；仅本机可访问 |

### 1.2 容器间直连（docker 内网，不对外）

| 容器名:端口 | 服务 | 被谁访问 | 用途 |
|---|---|---|---|
| `server:8000` | Django/Gunicorn | `web`(nginx) | 业务 API、Django Admin |
| `minio:9000` | MinIO API | `server`、`web`(nginx) | 文件上传（server 直连）、下载/预览（web 反代） |
| `minio:9001` | MinIO Console | （默认不暴露） | 仅容器内可访问 |

---

## 2. `5173` 端口路径说明

浏览器访问 `http://127.0.0.1:5173` 时，nginx 按路径分发：

| 路径 | 转发目标 | 说明 |
|---|---|---|
| `/` | 前端 SPA 静态产物 | `try_files` 兜底到 `index.html`，支持 vue-router history 路由 |
| `/api/v1/...` | `server:8000` | 业务 API（DRF） |
| `/admin/...` | `server:8000` | Django Admin 后台 |
| `/static/...` | 共享卷（collectstatic 产物） | Django Admin 等静态资源 |
| `/tfpc/...` | `minio:9000` | 文件下载/预览（预签名 URL 指向此路径） |

> 路由前缀 `/tfpc/` 对应 `MINIO_BUCKET=tfpc`。若修改了桶名，需同步修改 `web/nginx.conf` 中的 `location` 前缀。

---

## 3. 快速验证

在本机执行：

```bash
# 前端首页（200）
curl -I http://127.0.0.1:5173/

# 后端 API（401/403 均正常，表示服务在）
curl -I http://127.0.0.1:5173/api/v1/

# Django Admin（302 跳转登录，正常）
curl -I http://127.0.0.1:5173/admin/
```

查看端口监听：

```bash
# 应仅看到 127.0.0.1:5173 对外；8000/9000 不会出现在宿主机端口表
sudo ss -lntp | grep 5173
```

## 4. 后端 API 前缀（经 `/api/v1/` 访问）

API 基础地址：`http://127.0.0.1:5173/api/v1`

> 下文路径均省略基础地址，示例 `POST /auth/login/` 即 `POST http://127.0.0.1:5173/api/v1/auth/login/`。

### 4.1 认证与 Token

- `POST /auth/login/`
  - 获取 JWT：返回 `access` 与 `refresh`
  - 请求体：`{"username": "...", "password": "..."}`
- `POST /auth/refresh/`
  - 刷新 JWT
- `GET /auth/me/`
  - 获取当前用户信息（需 `Authorization: Bearer <access>`）

### 4.2 项目与成员

- `GET /projects/`：项目列表
- `POST /projects/`：创建项目（需系统管理员/项目管理员权限）
- `GET /projects/{id}/`：项目详情
- `PATCH /projects/{id}/`：更新项目
- `GET /projects/{id}/members/`：项目成员列表
- `POST /projects/{id}/members/`：添加成员

### 4.3 文件中心（核心）

`files` 相关接口均以 `/files/` 为前缀（带项目成员校验与 ACL 权限校验）。

- `GET /files/tree/?project=<projectId>&include_personal=0|1`：目录树
- `GET /files/?project=<projectId>&folder=root|<folderId>&include_personal=0|1`：目录列表
- `POST /files/folders/`：新建文件夹
- `POST /files/upload/`（multipart）：上传文件
- `GET /files/{id}/download/`：获取下载预签名 URL（指向 `127.0.0.1:5173/tfpc/...`）
- `GET /files/{id}/preview/`：获取预览预签名 URL
- `POST /files/{id}/update-content/`（multipart）：覆盖更新文件内容（产生版本记录）
- `GET /files/{id}/versions/`：历史版本
- `POST /files/{id}/rename/`：重命名
- `POST /files/{id}/move/`：移动
- `POST /files/{id}/delete/`：软删除（进入回收站）
- `GET /files/recycle/?project=<projectId>`：回收站列表
- `POST /files/{id}/restore/`：恢复
- `DELETE /files/{id}/purge/`：彻底删除
- `GET /files/search/?project=<projectId>&q=<keyword>&include_personal=0|1`：搜索

### 4.4 权限（ACL）

- `GET /files/acl/?project=<projectId>&target=root|<fileEntryId>`
- `POST /files/acl/grant/`
- `DELETE /files/acl/{acl_id}/`

### 4.5 审计

- `GET /audit/?project=<projectId>&action=<ACTION>`

### 4.6 系统备份（管理员）

- `GET /backups/`：列出备份文件
- `POST /backups/create/`：创建备份
- `POST /backups/restore/`：恢复备份

## 5. Django Admin

- `GET http://127.0.0.1:5173/admin/`
  - Django 自带后台，用于查看/管理数据库对象
  - 需用 `DJANGO_SUPERUSER_*`（或 `createsuperuser`）创建的超级管理员登录

## 6. 常见问题

### 6.1 前端能打开但登录失败

常见原因：前端构建时 `VITE_API_BASE` 未设为 `/api/v1`，导致浏览器请求到错误地址。

解决：确认 `.env` 中 `VITE_API_BASE=/api/v1` 后重新构建：

```bash
docker compose up -d --build web
```

### 6.2 访问 `http://127.0.0.1:5173/login` 404

本方案 nginx 已配置 `try_files $uri $uri/ /index.html` 兜底，SPA 路由不会 404。若仍 404，检查 `web/nginx.conf` 的 `location /` 块。

### 6.3 文件下载 403 SignatureDoesNotMatch

预签名 URL 的 `Host` 与后端 `MINIO_PUBLIC_ENDPOINT` 不一致。确认两者均为 `127.0.0.1:5173`，且 nginx `/tfpc/` 块保留原始 Host 头（`proxy_set_header Host $http_host;`）。

### 6.4 想从其他机器访问 5173

默认仅本机可访问。如需局域网访问，把 `docker-compose.yml` 中 `web` 的端口映射改为 `"5173:5173"`（监听所有网卡），并注意安全风险。
