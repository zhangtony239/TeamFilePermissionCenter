# 界面信息架构（含“回收站多入口”与“个人区开关”）

> 目标：减少不必要页面跳转；回收站入口多处可达但复用同一 API 与同一视图；提供“显示/隐藏个人区”的开关。

## 1. 页面/视图清单（MVP）

- 登录页
- 首页（Dashboard：概览/提醒/近期动态/统计）
- 项目管理页（左侧项目列表 + 右侧项目详情面板）
- 项目管理页（成员/角色）
- 项目文件页（核心：文件树/列表/预览/上传/操作）
- 团队管理页（成员库/角色模板/成员参与项目视图）
- 奖项库（全局奖项记录，可按项目/阶段过滤）
- 日历（赛程/里程碑视图，数据来自 Project Events）
- 任务（看板/甘特的数据维护与视图）
- 权限管理视图（可嵌入侧栏或抽屉；不强制新增独立页面）
- 回收站视图（同样可嵌入：不强制新增独立页面）
- 审计查询页（可选：MVP 可先简单列表）
- 系统设置页（用户/角色/权限、备份策略等）

> 明确省略：研发流程（PR/Issue/CI）、工作流(BPM)、插件/迁移/发布等开发者或平台能力页面。

## 2. 项目文件页（核心工作台）

### 2.1 主要区域
- 左侧：项目内目录树（可折叠）
- 顶部：路径面包屑（显示当前目录）
- 中间：文件列表（表格或卡片）
- 右侧：预览区域（PDF.js/图片/文本），也可新标签页

### 2.2 “个人区”开关
- 位置建议：文件列表工具栏（与搜索框同一行）
- 文案：`显示个人区`（开） / `隐藏个人区`（关）
- 行为：仅影响列表返回参数 `include_personal`，不改变权限。

## 3. 回收站多入口（同一 API）

### 3.1 入口位置（先多做，后续可删）
> 这些入口都跳到同一“回收站视图”，复用同一套 API。

- 入口 1：项目文件页工具栏固定按钮 `回收站`
- 入口 2：左侧导航（项目内菜单）`回收站`
- 入口 3：面包屑右侧快捷入口（图标/文字）`回收站`
- 入口 4：文件列表空状态（无文件时）的大按钮 `查看回收站`
- 入口 5：右键菜单（对任意文件/目录）里提供 `查看回收站`（可选）
- 入口 6：删除成功提示 Toast 内联链接 `去回收站`

### 3.2 回收站视图的功能（MVP）
- 列表：文件名、原路径、删除人、删除时间、到期时间
- 操作：恢复、彻底删除（可批量）
- 搜索：按文件名/路径关键词

### 3.3 API 复用点
- 列表：`GET /recycle/list`
- 恢复：`POST /recycle/restore`
- 清理：`POST /recycle/purge`

## 4. 文件/目录右键菜单（建议）

- 打开/预览
- 下载（按权限显示/隐藏）
- 上传到此处
- 重命名
- 移动
- 删除（进入回收站）
- 权限（打开 ACL 管理）

## 5. 降跳转策略（UI 交互建议）

- 回收站与 ACL 管理尽量做成“同页侧栏/抽屉”而不是新页面（可保留独立路由，便于直达）。
- 删除后提供 Toast：`已移入回收站` + `撤销/去回收站`。

## 6. 项目管理页（参考 NocoBase：左列表 + 右侧详情）

目标：点击项目后，右侧展示项目细节（编码/描述/成员/创建/起止日期/比赛阶段/进度/奖项）。

- 左侧：项目列表（Table）
	- 支持：搜索、按状态/阶段筛选（MVP 可先只做搜索）
	- 行点击：刷新右侧详情面板
- 右侧：项目详情面板（Details + Tabs）
	- 概览：进度条（0-100）、当前比赛阶段（校/市/省/国）、起止日期、描述
	- 团队成员：成员表（姓名/角色/是否在队）
	- 奖项：按阶段记录（可多条）
	- 赛程：里程碑/事件（后续可做日历/时间线）
	- 文件：跳转/嵌入项目文件中心
	- 审计：跳转/嵌入审计查询
